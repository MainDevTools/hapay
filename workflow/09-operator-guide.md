# Оператор-гайд: як керувати Claude Code

**Статус: чернетка.** Доповнює [08-interaction-protocol.md](08-interaction-protocol.md) з **вашого** боку. Протокол — правила для обох; тут — **ваш playbook**, бо дисципліни CC (я це довів) недостатньо: **ви — гальмо й арбітр.**

## Ваші 8 ходів (коли й навіщо)

### 1. Задавай рамку в запиті
CC вгадує, якщо не сказано. Дай: **навіщо** (мета), **рівень** (чернетка / вивірене / фінал), **бюджет** (час/обсяг), **формат** (список / док / код / рішення). Один рядок економить хід переробки.

### 2. Тисни «стоп / досить»
**CC не зупиниться сам надійно** — це доведений режим відмови (`02-invariants` E-bis). Кажи «стоп/досить/пауза», коли відчуваєш: churn, повтори, «ще один список», спадну цінність. Не чекай, поки CC сам загальмує.

### 3. Вимагай пушбек — і калібруй його
Скажи прямо: **«не згоджуйся, якщо маєш підстави»**. Коли CC не згоден — це **сигнал перевірити**, не автоматично «CC правий» і не «CC впертий». Питай «чому» → оціни аргумент. Anti-sycophancy працює, лише якщо ти його **хочеш** і показуєш це.

### 4. Не приймай «я зробив» без evidence
Вимагай **дифф / тести / вивід `/verify`**. Для зовнішніх фактів (крамниці, конкуренти, закони) — **«перевір із першоджерела»**: CC двічі помилявся з одного джерела (Pethouse, Lume). Правило ≥2 підтверджень.

### 5. Аудитуй CC (не довіряй наосліп)
Ти **не можеш** незалежно перевірити мене, не переробивши. Тож для **критичного**: попроси показати дифф; **переrun ключову перевірку сам** (grep/тест); не покладайся лише на мій Bash-вивід для того, що дорого помилитись. Довіра — до дрібного; аудит — до важливого.

### 6. Розпізнавай sunk-cost
Коли **зростання опису** відчувається як прогрес, а коду нема — це **пастка** (кожне додавання виправдовує наступне). Сигнал: «ми доопрацьовуємо N-й раз». Реакція: **перемкни на виконання** (код/рішення), не на N+1 документ.

### 7. Тримай рішення за собою
**🧭 = твоє** (топологія, монетизація, гроші, юр-згода). Не делегуй стратегію CC. Закривай відкриті рішення (`03-decisions` O1–O9) **явно** — CC їх не вирішить і не має.

### 8. Знай, коли перемкнути модальність
Чат — не єдиний режим. Іноді найкорисніше: **закрити чат → запустити код / прочитати артефакт / вирішити офлайн**. Нескінченне шліфування в чаті — теж пастка (пор. п.6).

## Швидка діагностика «щось іде не так»

| Симптом | Твій хід |
|---|---|
| CC генерує ще список, а тобі не легшає | «стоп» (п.2) + перемкни на код/рішення (п.6) |
| CC каже «готово/працює» без доказів | вимагай evidence (п.4) |
| CC надто погоджується | «не згоджуйся, якщо маєш підстави» (п.3) |
| Не знаєш, чи CC перевірив факт | «перевір із першоджерела» + переrun сам (п.4/5) |
| 40 ходів, 0 коду | sunk-cost (п.6) → Worker-режим |
| CC чекає на твоє рішення | закрий O-пункт явно (п.7) |
