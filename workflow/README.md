# Взаємодія «директор ↔ Claude Code» (робочий процес)

**Статус: чернетка, доробляємо.** Ця папка описує, як ведеться розробка «Радара знижок» у моделі *соло-директор + Claude Code як основний розробник*, зокрема патерн **дві сесії: Диригент + Виконавець**.

Це **не** частина технічної специфікації (`../00-index.md` … `../15-hlosarii.md`) — це опис *процесу*, а не продукту.

## Розподіл відібраного по файлах (P8 + P9)

| Файл (створити) | Що містить |
|---|---|
| **`04-captain-log.md`** | Стан проекту: де ми, що зроблено, що в роботі, блокери. **Shutdown-ритуал** (№17) дописує сюди наприкінці сесії. Ціль **START HERE** (№50) веде сюди. |
| **`05-bootstrap-prompts.md`** | Стартові промпти Диригента й Виконавця; **context-budget** (№3 — що саме вантажити, не всі 16 розділів). |
| **`06-backlog.md`** | Задачі S0…SN. Поля: **розмір** S/M/L «чи влазить у сесію» (№7), **залежності** (№8), **стан** draft→ready→in-progress→review→done→merged (№9); **WIP=1** (№10). |
| **`07-conventions.md`** | repo-map; **Definition of Ready**; протокол **«застряг»** (BLOCKED+стоп); **механізм реальність-фідбеку**; **стандарт рев'ю** + обов'язкове **людське рев'ю юр-інваріантів §7.4 і §5** (№25); **git-конвенції** + **CI-гейт** (№28) + **порушення=блок мержу** (№29); як оновлюється спека; правило **≥2 підтвердження з першоджерела** (№44). |
| **`02-invariants.md`** (доповнити) | Які інваріанти стають **тестами першими** (№26: A1/A2/B6/C9/D11); **pre-commit** хуки (№27); посилити E14 (≥2 підтвердження, №44). |
| **`CLAUDE.md`** (корінь, коли код) | Дистилят інваріантів + перелік **guardrail-тестів** (№26–29). |
| **`README.md`** (цей) | **START HERE** (№50) — єдина точка входу для нової сесії: README → captain-log → CLAUDE.md → backlog. |
