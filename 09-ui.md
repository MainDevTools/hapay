# Розділ 9. Інтерфейс

Головна цінність UX — те, чого немає в конкурентів: **знижки-first + категорії + фільтри + бейдж-верифікація** в одному екрані. Формат — десктоп, але кожен екран проєктується як замінна презентація над Шаром 2 (розділ 8).

## 9.1. Головний екран — категорійний перелік знижок

```
┌─ Радар знижок ─────────────────────────────────────────────────────┐
│ Категорія: [Сухий корм · коти ▾]   Крамниця: [усі ▾]                │
│ Бейдж: [✓ підтверджені] [▷ заявлені] [⚠ від завищеної]  ≥ [10%] ▾  │
│ Сорт: [верифікований % ▾]                              🔍 [пошук]   │
├────────────────────────────────────────────────────────────────────┤
│ ✓ Royal Canin Kitten 2кг   890→620 грн   −30%  MasterZoo   [деталі]│
│    ✓ підтверджено: ціна за 30 дн не була нижчою за 885 грн          │
│ ▷ Purina One 1.5кг         520→410 грн   −21%  Petslike    [деталі]│
│    ▷ заявлено крамницею · перевірка через 12 днів                   │
│ ⚠ Josera Kitten 2кг        1200→720 грн  −40%(заявл.)  Pethouse    │
│    ⚠ реально −4%: ціну підняли до 750 за 9 днів до акції            │
├────────────────────────────────────────────────────────────────────┤
│ 128 знижок · 6 крамниць · оновлено 12:05                            │
└────────────────────────────────────────────────────────────────────┘
```

> *Мокап ілюстративний: ціни показано округлено для стислості — реальний UI виводить 2 знаки (`620,50 грн`, §9.5); бейдж ✓ має підстан ✓⁻ «попередній» (§5.3).*

- **Категорія** — канонічна ієрархія (§2.6); дефолт — обрана вертикаль (`app_config('default_category')`).
- **Фільтри:** крамниця (`source_id`), стан бейджа (`discount_event.badge_state`), мінімальний % (`verified_pct`), наявність (`price_snapshot.in_stock` останнього снапшота).
- **Сорт:** верифікований % / заявлений % / ціна (`current_kop`) / свіжість (`computed_at`).
- Кожен рядок несе бейдж (§5.3) + **однорядкове фактологічне пояснення** (§5.4 / ст.277 ЦК). Зелений має підстан **«попередній»** (`verified_provisional`, §5.3) — візуально ✓ з приміткою «мало точок історії, може уточнитись», щоб перші тижні на desktop-каденсі не виглядали порожніми; фільтр стану бейджа розрізняє повний і попередній зелений.
- **Пошук (🔍)** і збережені запити (`watchlist.kind='query'`) — FTS5 над `store_product.title` (`store_product_fts`, §6), звужені поточним фільтром категорії/крамниці. Текст користувача **санітизується** перед `MATCH` (спецсимволи FTS5 `" * : ( )` квотуються або екрануються — інакше назва товару з лапками/зірочкою кидає помилку синтаксису запиту).
- **Мапінг рядка на дані:** `discount_event` (`current_kop`/`old_declared_kop`/`declared_pct`/`verified_pct`/`reference_kop`/`badge_state`) + `store_product.title`/`image_url` (мініатюра поруч рядка, hotlink/blurhash — §9.2) + `source.name`. Багатша хронологія накачки (⚠: коли/на скільки підняли ціну) — **derived** із `price_snapshot`, не окреме поле.

## 9.2. Картка знижки (деталі)

```
┌─ Royal Canin Kitten 2кг · MasterZoo ───────────────────────────────┐
│ Зараз: 620 грн    Заявлена стара: 890 грн (−30%)                    │
│ ✓ ПІДТВЕРДЖЕНО ІСТОРІЄЮ                                             │
│   30-денний мінімум: 885 грн → реальне зниження −30%                │
│                                                                    │
│ Історія ціни (60 днів):        ┌──────────────────────────┐        │
│                                │ 890───────────╲___620     │        │
│                                └──────────────────────────┘        │
│ Заміри: 1–2×/добу · 118 точок · джерело: JSON-LD                   │
│ [Відкрити в крамниці ↗]   [Стежити за товаром ☆]                   │
└────────────────────────────────────────────────────────────────────┘
```

Показуємо: **фото товару** (`store_product.image_url`, §6.3) — рендер поверх `image_blurhash`-фону (миттєвий, елегантний фолбек при битому hotlink; `onerror`→плейсхолдер), джерело `feed` (право показу) або `hotlink` (вантажиться з CDN крамниці, ми байти не тримаємо — §7.4); поточну + заявлену стару, юніт-ціну грн/кг де крамниця її друкує (§2.7, M2+), верифікований бейдж із поясненням, **графік історії ціни** (компонент: легкий **inline-SVG** з `price_daily` §6.3 — 30–90 днів, лінія + позначка 30-денного мінімуму й `announce_date`; без важких chart-бібліотек, щоб Mini App лишався легким §9.7; backfill-точки — пунктиром), provenance (частота/кількість замірів `price_snapshot`, метод `source_method`; **ретро-набудовані точки** `is_backfill` позначені окремо — нижча довіра, §10.7). Кнопка «відкрити в крамниці» — прямий URL (не проксі) у M1. **Перехід до монетизації (M4, §4.5):** якщо дохід піде з афілійованих deeplink-ів, ця кнопка стане точкою **rewriting URL** через партнерську мережу — тоді потрібен окремий rewriting-шар (мапінг store→affiliate-параметри) **і обов'язкове розкриття** «партнерське посилання» поруч (інакше — недобросовісна практика, §7). Архітектурно закладаємо, що презентація вже має хук на URL (легко ввімкнути rewriting без переробки), але сам rewriting і дисклеймер — поза M1.

## 9.3. Інбокс / нові знижки (щоденний огляд)

Дефолтний екран після нічного збору — пріоритезований список нового з дельти. **Скоуп:** у M1 інбокс працює на дефолтній категорії без пушів; секція 2 і сповіщення потребують watchlist — це M2 (§11.4):

| Пріор. | Секція | Дія |
|---|---|---|
| 1 | ✓ Нові підтверджені знижки у моїх категоріях | «У перелік» |
| 2 | ☆ Watchlist: товар/категорія подешевшали *(M2)* | «Відкрити» |
| 3 | ⚠ Викриті накачані «−50%» (жовті) у моїх категоріях | «Показати» |
| 4 | ⚙ Джерело замовкло (адаптер зламався / заблокували) | «Перевірити» |

**Анти-повтор — по СТАНУ бейджа, не по події (щоб не глушити апгрейди).** Показане логується в `alert_log` за ключем `UNIQUE(watchlist, discount_event, badge_state)` (§6.5). Наслідок: коли бейдж **апгрейдиться** `declared → verified_provisional → verified` (товар нарешті **довів** реальну знижку — найцінніший для користувача момент) або `→ pumped` (знижку **викрито** як фейк — теж вартий сповіщення для watchlist), це **новий стан** → одне нове сповіщення; а той самий стан повторно **не блимає**. Правило переходів: сповіщаємо на вхід у `verified`/`verified_provisional`/`pumped`, ще не залогований для цієї події; **downgrade** verified→provisional (розрідження історії) і закриття події — **не** сповіщаємо (шум). Так інбокс ловить дві найважливіші події життя знижки (підтверджено / викрито), а не лише першу появу.

## 9.4. Адмін-екран джерел і конфігу

Для власника: перегляд/редагування `source` (крамниці, каденс, `frozen_at`), `source_category_map` (мапінг категорій), `detection_config` (поріг/вікно). Плюс здоров'я адаптерів: останній `scan_run.status`, `parse_success_rate`, коли востаннє була валідна ціна. Червоний індикатор, якщо `parse_success_rate` впав або джерело заморожене.

## 9.5. Доступність і локаль

- Одномовний UA (MVP); locale-правила — в `app_config` (`ui_decimal_sep=','`, `ui_thousands_sep=' '`, `tz_display='Europe/Kyiv'`).
- Ціни — UA-формат (`1 250,50 грн`); дати показу — Київ. **Конверсія `*_kop` → грн — завжди 2 знаки** (`62050 → 620,50 грн`), не ціле: корм часто коштує X,99, і показ цілим розійшовся б з реальною/експортованою ціною. Мокапи §9.1/§9.2 показують цілі суми лише для стислості.
- Клавіатурна навігація на списку/фільтрах; **жоден сигнал не лише кольором** — бейдж дублюється іконкою+текстом (✓/▷/⚠), не лише зеленим/жовтим (CVD-safe).
- Експорт списку — CSV (UTF-8 з BOM, інакше Excel-UA ламає кирилицю).

## 9.6. Онбординг (перший запуск)

Перший запуск — короткий локальний майстер, без реєстрації:

1. **Вибір вертикалі** — «Зоотовари» (MVP; електроніка — далі). Ставить `app_config('default_category')` й активує **вбудовані адаптери** цієї вертикалі: 6 крамниць уже сконфігуровані в постачанні (base_url, `discount_url`, селектори каскаду §4, `source_category_map`, канарки §10.9) — власник **нічого не налаштовує вручну**.
2. **Чесний таймлайн історії.** Дефолт — **живий збір з нуля**: банер «Заявлені знижки — вже сьогодні; перевірені історією — за ~4 тижні». Опція «спробувати архів (§4.6)» — увімкнена, але з чесним попередженням: для зоо-крамниць покриття Wayback майже нульове, тож архів — бонусні точки історії, а не обіцянка зелених бейджів з дня 1.
3. **Перший збір** — прогрес «Збираємо знижки… (N/6 крамниць)»; щойно є дані — заявлені знижки (Стадія A) на головному екрані.
4. Готово → §9.1; планувальник ставить каденс (§10).

**Стани порожнечі / cold-start (щоб не виглядало зламаним):**

| Стан | Що показуємо |
|---|---|
| До першого scan | «Перший збір триває… (N/6)» — прогрес, не порожній екран |
| Зібрано, < 30 дн | знижки ▷заявлені + банер «Перевірка історією за ~N днів» |
| Архів-спроба триває | «Шукаємо архівні ціни… що знайдеться — додамо в історію» (без обіцянки зелених бейджів — §4.6) |
| Пропущені скани (ПК був вимкнений/спав) | банер «Історія має прогалини за N дн — точність перевірки знижена»; catch-up-скан при старті (§10.10) |
| Джерело замерзло на старті | «Крамниця X недоступна — решта працює» (не блокуємо весь екран) |
| **Boot-gate не пройдено** (бракує config-дефолту) | екран «Налаштування незавершене: <що саме>» + «Полагодити» — **не крах** |

**Очікування чесно з першого екрана:** банер пояснює двостадійність (заявлено зараз → підтверджено історією потім), інакше нейтральні/сірі бейджі перших днів читаються як «не працює».

## 9.7. Telegram Mini App — мобільний фронт (центральний, ухвалено §8.10)

Мокапи §9.1–9.2 — **desktop-рендер**. Рекомендований фронт central-light — **Telegram Mini App** (мобільний web усередині Telegram), той самий Шар 2, інша розкладка. Тут — макет і флоу, бо диференціатор «category-first» не можна лишати лише desktop-прозою.

**Головний екран (мобільний, вертикальний):**
```
┌─ Радар знижок ──────────────┐
│ 🔍 Пошук        [Сухий корм▾]│  ← категорія — велика, зверху (category-first)
│ [✓][▷][⚠]  ≥10%▾  сорт: %▾  │  ← бейдж-чипи + фільтри одним рядком
├─────────────────────────────┤
│ ┌───┐ Royal Canin 2кг       │
│ │IMG│ 890→620  −30% ✓        │  ← мініатюра (hotlink §9.2) + бейдж
│ └───┘ MasterZoo · 885 за 30д │
│ ┌───┐ Purina One 1.5кг      │
│ │IMG│ 520→410  −21% ▷        │
│ └───┘ Petslike · перевірка…  │
├─────────────────────────────┤
│ 128 знижок · оновлено 12:05  │
└─ [Стрічка] [Watchlist] [☰] ─┘  ← таб-бар знизу (мобільна навігація)
```

**Флоу (3 екрани, не більше — мобільна простота):**
1. **Стрічка** (головна) → тап на рядок → **Картка** (§9.2: фото, графік історії, провенанс, «відкрити в крамниці»);
2. **Watchlist** (☆) → категорії/товари під стеженням; додати з картки;
3. **Меню (☰)** → категорія, налаштування сповіщень (§9.3), «як ми перевіряємо» (методологія — довіра, R-P1-9/§16).

**Мобільна специфіка (чого нема в desktop-§9):**
- **Тема Telegram** — Mini App успадковує світлу/темну тему клієнта (`themeParams`); не хардкодимо кольори (бейджі лишаються CVD-safe §9.5 в обох темах).
- **Нативна навігація** — Telegram BackButton/MainButton замість власних кнопок; таб-бар знизу (палець).
- **Category-first на малому екрані** — категорія-селектор **зверху й великий** (це диференціація проти лінійного Lume-фіду); фільтри — компактні чипи, не таблиця.
- **Нуль інсталяції** — відкривається з бота/каналу; `initData`-авторизація (§8.10.1), watchlist прив'язаний до `telegram_user_id`.
- **Фото — hotlink у web-Mini-App** (§7.4/§8.10), не `send_photo` ботом.
- **Пуш — через бота** (§9.3): нова знижка / апгрейд бейджа → повідомлення з кнопкою «Відкрити в Mini App».

**Бот (текстовий, доповнює Mini App):** команди `/search корм`, `/deals` (топ дня), `/watch`; складний перегляд — завжди в Mini App (лінійність бота компенсує category-first вітрина). **Свідомо відкладено:** піксель-макети, дизайн-система, юзабіліті-тест — до валідації попиту (S0.2); тут — рівень «зрозуміло, що будувати».
