### 1. Ціль
Побудувати `probe.py`, який (1) звіряє **поверхню** 6 зоо-крамниць, (2) рахує **ліквідність** (унікальні знижені позиції/добу), (3) запускається щодня в GitHub Actions і **накопичує** дані, щоб за ~3–4 тижні виміряти badge-rate (S0.3). Це монітор+лічильник, **не** конвеєр.

### 3. Definition of Done
- [ ] `probe.py` для 6 крамниць друкує per-store: HTTP-код, тир (A/B/C), КБ, #карток із ціною, #`old-price`, наявність JSON-LD, anti-bot-маркер;
- [ ] рахує **унікальні знижені позиції/добу** на 2 робочих (Pethouse, PetChoice) — дедуп за канонічним `external_ref` (§4.8);
- [ ] **≥2 прогони підряд узгоджені** (проти артефакту, урок Pethouse) — розбіжність > поріг → флаг, не висновок;
- [ ] **Перевірка персоналізації ціни (R-P0-4/#101):** той самий товар запитати **без куки / з різними куками / (за можливості) різних регіонів** → чи ціна **стабільна**. Це відповідає, чи «ціна, яку ми збираємо» = «ціна, яку бачить юзер» — критичне для передумови продукту. Результат: «стабільна / персоналізована» на джерело;
- [ ] запускається в **GitHub Actions cron** (1×/добу), результат у CSV/JSONL, комітиться в репо (історія поверхні);
- [ ] базові тести на парсер (на VCR-касеті хоча б однієї крамниці);
- [ ] evidence у §Outcome: вивід за 2 дні + дифф.

### 4. Поза скоупом
Адаптери в БД, `detect_pass`, UI, схема `0001` — **ні**. Тільки probe+лічильник+CI. Не воювати з CAPTCHA-крамницями (Zootovary) — позначити тир C і йти далі.

### 5. Guardrails
- Не обходити CAPTCHA/техзахист (§7.4); прийняти авто-куку — можна.
- Ввічливий rate + джиттер, `max_req_per_min_per_host` (§10.2).
- **≥2 підтвердження** перед будь-яким висновком «крамниця X віддає/не віддає» (E14).
- Зберігати лише факт (числа), не байти сторінок у репо (крім касет-фікстур).

### 6. Підказка щодо кроків
Реальні акції-URL уже знайдені (§3.3 зоо, §3.4 електроніка) → fetch (httpx) → рахунок класів/JSON-LD → дедуп `ext_ref` → вивід. GitHub Actions: `.github/workflows/probe.yml`, cron, `actions/checkout` + `uv` + коміт результату.
