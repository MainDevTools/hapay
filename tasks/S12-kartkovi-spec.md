# Задача S12 — Характеристики з карток товарів (повні спец-таблиці)

**Роль:** Виконавець (код) · **Автор:** Диригент, 2026-07-24 · **Статус:** `в роботі` (оператор 2026-07-24: «формулювання ок, скоуп 2+, стартуй у вільних слотах» — бекфіл на всі 4467 груп 2+)
**Залежності:** матчинг (match_key-групи), черга collect_task, /api/ingest/html, MAUI-картка групи.
**Рішення оператора 2026-07-24:** з трьох шляхів (назви / картки / чужі бази) обрано **картки товарів**.

### 0. Навіщо
Групова картка порівнює ціни, але не каже, ЩО це за товар (діагональ, пам'ять, вага…).
Повні характеристики лежать у спец-таблицях карток крамниць — ми їх зараз не збираємо
(збираємо лише лістинги). Одна картка на групу закриває специфікації для всіх крамниць
групи одночасно.

### 1. Юр-рамка (розширення інваріанта B — підтвердити формулювання)
- До списку «зберігати лише факти» (ціна/назва/URL/час) додаються **характеристики:
  пари назва-значення технічних фактів** («Діагональ екрана» → «43"»). Факти не
  охороняються авторським правом — та сама природа, що ціна.
- **ОПИС (зв'язний маркетинговий текст) НЕ зберігаємо ані байта** — авторський текст,
  аналог фото-правила. Сирий HTML картки парситься і відкидається (як лістинги нині).
- Фото-правило без змін (лише вказівники). Чужі бази специфікацій (GSMArena тощо) — 🔴.
- Провенанс завжди: кожен spec-набір зберігає source + URL картки + дату збору.

### 2. Межі v1
**In:** одна картка на крос-групу (специфікації спільні для групи); бекфіл груп із
**3+ джерелами (зараз 2404)**; нові групи — у міру появи; секція «Характеристики»
в MAUI-картці. **Out:** групи-одиначки (нема порівняння — цінність специфікацій нижча
за ціну збору); повторний збір (специфікації не протухають — разово на товар);
фільтри/фасети за характеристиками (v2); описи-тексти (назавжди).

### 3. Обсяг і пропускна здатність (чесні цифри, 2026-07-24)
Крос-груп 2+ джерела: **4467**; 3+: **2404**; 4+: **1393**. Збирач у дефіциті ~10×
(≈138 стор/добу факт), тому card-задачі мусять **підбирати лише вільні слоти**
(пріоритет нижчий за page/sitemap): при ~30-60 карток/добу бекфіл 2404 груп ≈
**40-80 днів**. Це чесна швидкість за нинішньої стелі; прискорення = рішення 🧭 по
звіту A–E (окремо, не блокує старт).

### 4. Механіка
- **Черга:** нова `kind='card'` у collect_task — **одноразова** (виконалась → закрилась,
  без repeat; повтор лише вручну). Постановник: бекфіл-скрипт + гачок «група стала 3+».
- **Вибір крамниці-джерела в групі:** пріоритет за (card-адаптер є → mode=fetch
  дешевший за render → покриття). Перші кандидати за покриттям груп: Allo 1988,
  Foxtrot 1877, Citrus 1628, Rozetka 1447 (розвідка електроніки 07-24: картки
  віддаються звичайним GET — але серійний burst-тест обов'язковий, вердикти протухають).
- **Збір:** телефон тягне URL картки (fetch/render за mode крамниці) → `/api/ingest/html`
  → сервер парсить новим методом адаптера `parse_card(html)` → спец-таблиця.
- **Robots:** картки можуть мати ІНШІ правила, ніж лістинги — перевіряти по UA-секціях
  власним матчером ДО додавання крамниці (межа типу «ціни-картинки 130» — тверда).

### 5. Схема (міграція NNN)
```sql
product_spec(spec_id BIGSERIAL PK, store_product_id BIGINT FK UNIQUE,  -- картка-джерело
             source_url TEXT NOT NULL, collected_at TIMESTAMPTZ NOT NULL DEFAULT now())
spec_attr(spec_id BIGINT FK, position INT, name TEXT NOT NULL, value TEXT NOT NULL,
          PRIMARY KEY (spec_id, position))
```
Прив'язка до store_product_id (не match_key — він мутує при бекфілах mpn); показ у
групі — через match_key картки-джерела на момент запиту.

### 6. API + MAUI
- `GET /api/product/{store_product_id}/specs` → `{specs: {source, source_url,
  collected_at, attrs: [{name, value}]} | null}` (окремий endpoint — /offers і /choice
  незмінні). Для будь-якого члена групи віддається spec-набір групи.
- MAUI: секція «Характеристики» в DetailPage (expander, перші 6 рядків + «показати всі»),
  підпис-провенанс «за даними {крамниця}, {дата}».

### 7. Definition of Done
- [ ] Розвідка + серійний burst карток 2-3 крамниць-кандидатів; robots по UA-секціях.
- [ ] `parse_card` для ≥2 крамниць: касети з РЕАЛЬНИХ карток + golden-тести
      (пастки: розстрочка-ціна, «характеристики» в табах/JS, nbsp, одиниці виміру).
- [ ] Міграція + черга kind='card' (одноразовість; не голодує page-збір — тест).
- [ ] ingest_html: card-гілка (та сама автентифікація/валідація; опис-текст не
      персиститься ніде — негативний тест).
- [ ] API + тест round-trip; MAUI-секція + синк + збірка 0 errors.
- [ ] Evidence: жива група з характеристиками наскрізь (телефон → сервер → API → скрін).

### 8. Guardrails
- Тільки пари назва-значення зі спец-таблиці; жодного зв'язного тексту (навіть у note).
- Одна картка на групу; card-задачі лише у вільні слоти (page-збір першим).
- Серійний burst перед «крамниця береться» (личини блоку: 200+0KB, staging-шелл).
- Розмітка/вердикти протухають — розвідку не переносити з минулих сесій без перевірки.
