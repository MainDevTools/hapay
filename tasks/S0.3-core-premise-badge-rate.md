# Задача S0.3 — Валідація ядра: фейк-частка + чи з'являються бейджі

**Роль:** Виконавець (аналіз/скрипт) + ви (звірка) · **Автор:** Диригент, 2026-07-10 · **Статус:** `blocked-until` S0.1 накопичить дані
**Валідує:** R-P0-2 (#70, #97 — фейки є, формула ловить, зелені бейджі з'являються). **Залежності:** S0.1 (дані probe).

### 1. Ціль
Виміряти на **реальних даних**, чи справджується ядро продукту: (1) яка частка знижок фейкові; (2) чи формула §5 їх позначає жовтим; (3) чи з'являються зелені бейджі взагалі. Це **go/no-go гейт** — якщо ні, це пивот (§12.4), не провал.

### 2. Контекст
- `../05-detektsiya.md` §5 весь (Стадія A/B, стани бейджа, §5.7 числа);
- `../01-vstup.md` §1.1 (60–70% фейків — теза перевіряється тут);
- `../11-mvp.md` §11.3 (критерії успіху: ≥30% отримують бейдж; помітна частка жовтих);
- `../16-reyestr-ryzykiv.md` R-P0-2.

### 3. Definition of Done
- [ ] **(a) швидкий сигнал за день** — ручна вибірка ~100 знижок із S0.1 + звірка історії (probe-архів / скріншоти / Lume-check §12.2) → **груба фейк-%**;
- [ ] **(b) відкладений** — після ~3–4 тижнів даних S0.1: прототип `detect_pass`-логіки (§5, можна скрипт, не повна БД) рахує частку **зелений / жовтий / сірий** на реальній історії;
- [ ] **(c) go/no-go** проти §11.3 із числами: чи ≥30% отримують не-сірий бейдж; чи є помітна частка жовтих (викритих фейків);
- [ ] висновок: **будувати далі / пивот на «викривач фейків» (жовтий-first) / стоп** — з обґрунтуванням.

### 4. Поза скоупом
Повний конвеєр у продакшн-БД (прототип-скрипт достатньо); UI; інші вертикалі.

### 5. Guardrails
- **≥2 підтвердження** фейк-частки (не з одного джерела — урок Lume/Pethouse, E14).
- Фактологічність (§5.4): не публікувати оціночне про конкретну крамницю в цьому дослідженні.
- Статутний `reference` — з сирої історії probe, не з заявленої крамницею ціни (§5.6).

### 6. Підказка щодо кроків
Спершу (a) вручну за день — швидкий сигнал, чи взагалі є сенс чекати 3–4 тижні. Якщо фейків мало вже на вибірці — раннє попередження. (b) — коли S0.1 назбирає ≥ `min_reference_points` точок на товар.
