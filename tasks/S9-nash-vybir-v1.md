# Задача S9 — «Наш вибір» v1: найвигідніший спосіб купити (не найдешевша ціна)

**Роль:** Виконавець (код) · **Автор:** Диригент, 2026-07-24 · **Статус:** `draft` (чекає затвердження ваг оператором)
**Залежності:** S3 (discount_event — джерело індексу чесності), наявний персист/API груп. **Продуктова фішка.**

### 0. Фішка (навіщо)
Агрегатори порівнюють цифру ціни. Ми порівнюємо **повну вигоду покупки**: ефективна ціна
(з доставкою), чесність знижок крамниці (наша власна детекція — диференціатор, якого нема
ні в кого), наявність, самовивіз. У картці групи — блок:

> 🏆 **Наш вибір: Крамниця X** · Ви економите 1 250 грн проти найдорожчого варіанта.

### 1. Ціль
Сервер рахує для кожної (крос-група × крамниця) прозорий **score** і віддає в API
`our_choice` + складники; MAUI рендерить блок «Наш вибір» + бейдж чесності крамниці.
Жодного нового збору: все з наявних даних + два ручні довідники.

### 2. Межі v1
**In:** score з наявного (ціна, історія/бейджі, in_stock, індекс чесності) + довідники
`delivery_rule` і `store_network` (ручний сів міграцією) + рядок економії.
**Out (свідомо):** 💳 кешбек (🧭 монетизація — окреме рішення оператора); 🛡️ гарантія
(потребує КАРТКОВОГО збору — заблоковано дефіцитом збирача, див. звіт A–E); 📍 гео-«поруч»
(v1 — лише «мережа має самовивіз», без відстаней); 🤖 LLM (не потрібен: формула прозора й
пояснювана — це юр-перевага, не компроміс).

### 3. Формула (draft — ваги затверджує оператор, як пороги детекції §5)
Кандидати: рядки групи з `in_stock=true` (відсутній товар — не кандидат, це фільтр, не вага).

```
effective_kop = price_now_kop + delivery_kop(source, price_now_kop)
    де delivery_kop = 0, якщо price ≥ free_from_kop, інакше base_delivery_kop (довідник)

price_score   = (max_eff − eff) / (max_eff − min_eff)      # 1.0 = найдешевший ефективно; група з 1 крамниці → 1.0
honesty_score = 1 − pumped_share(source)                    # частка pumped серед discount_event крамниці за 90 днів,
                                                            # згладжування Лапласа (α=1): (pumped+1)/(events+2)
pickup_bonus  = 0.05, якщо store_network.has_pickup

score = 0.70·price_score + 0.25·honesty_score + pickup_bonus
our_choice = argmax(score);  savings_kop = max(effective_kop) − effective_kop(our_choice)
```
Пояснюваність — вимога: API віддає СКЛАДНИКИ, не лише підсумок. Ваги — у
`detection_config`-стилі (таблиця/конфіг, не хардкод), змінюються без деплою коду.

### 4. Схема даних (міграції NNN+)
```sql
delivery_rule(source_id PK→source, free_from_kop BIGINT, base_delivery_kop BIGINT,
              np BOOL, courier BOOL, note TEXT, updated_at)
store_network(source_id PK→source, has_pickup BOOL, cities_note TEXT, updated_at)
choice_weights(w_price NUMERIC, w_honesty NUMERIC, pickup_bonus NUMERIC,
               valid_from, valid_to)          -- патерн detection_config
```
Сів довідників — окремою data-міграцією з чесними значеннями по 28 крамницях
(джерело: сторінки умов доставки крамниць; [здогад] заборонено — лише виписане з їхніх
сторінок, з датою звірки в note).

### 5. Definition of Done
- [ ] `choice/core.py` — чисті функції: `effective_kop`, `pumped_share` (Лаплас), `score`,
      `pick(group_rows, rules, weights)` → (our_choice, savings, складники). Гроші — цілі
      копійки; ваги — Decimal; ділення на нуль (група з 1 крамниці, події=0) покриті.
- [ ] Синтетичні golden-тести: домінація ціни; перекидання вибору чесністю (дешевша, але
      pumped-крамниця програє при різниці < порога); free_from-межа доставки; група-одиначка;
      відсутність delivery_rule (fallback: доставка 0 + прапорець `no_delivery_data`).
- [ ] `detect`-незалежність: score НЕ пише в discount_event і НЕ чіпає формулу бейджів (§5).
- [ ] API: у відповіді групи — `our_choice {source, effective_kop, savings_kop, score,
      components{price_score, honesty_score, pickup_bonus, delivery_kop}}` + по кожній
      крамниці `honesty_score`. Незмінність наявних полів (сумісність MAUI).
- [ ] Тест API round-trip проти тест-БД (патерн test_api.py, TEST_DATABASE_URL).
- [ ] MAUI: блок «🏆 Наш вибір» у картці порівняння (крамниця, економія, tap → складники),
      бейдж чесності поряд із назвою крамниці. ⚠ синк `app-maui/` → VS-проєкт
      (`sync-maui.ps1`) перед збіркою.
- [ ] Дисклеймер у блоці (текст — оператор): «Розрахунок за відкритою формулою з публічних
      цін; крамниці не платять за позицію».

### 7. Guardrails
- Індекс чесності — З НАШИХ discount_event (сирі факти), ніколи з чужих рейтингів
  (Hotline/price.ua — 🔴 юр-постура).
- «Наш вибір» — рекомендація за формулою, не реклама: складники завжди видимі, ваги
  публічні. Формула/ваги — під людським рев'ю (аналог інваріанта C).
- Довідники — лише виписане з першоджерел (сторінки умов крамниць) з датою; протухле
  (updated_at > 60 днів) → API маркує `stale_delivery_data`, UI ховає доставку зі скора.
- Групи з 1 крамниці: блок не показуємо (нема з чим порівнювати) — лише історія/бейдж.
