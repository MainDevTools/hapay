# CLAUDE.md — «Радар знижок»

Категорійний, покрамничний детектор **справжніх vs накачаних** знижок в українських онлайн-крамницях. Перевіряє знижку статутною формулою **найменшої ціни за 30 днів** (Omnibus / закон №3153-IX). Фаза: специфікація → перший код. Мова проекту — українська.

> Це авто-завантажений вхід сесії. Він **дистилят** — повне живе в `workflow/` і `00-index.md`. Тримати лаконічним.

## Тверді інваріанти (не порушувати; повне — `workflow/02-invariants.md`)

**A. Гроші й дані**
- Гроші — **`BIGINT` копійки**, ніколи float (Postgres відхиляє не-ціле на рівні типу).
- `price_snapshot` — **append-only** (тригер `RAISE EXCEPTION` на UPDATE/DELETE + `REVOKE`); історія незмінна. Звичайна таблиця + покривний індекс `ix_ps_prod_window`.
- `reference_kop` (30-денний мінімум) — **рахувати з сирого `price_snapshot`** (не з агрегату).
- БД — **чистий PostgreSQL 16** (§6, ship на **Neon** free — T11); FTS = `tsvector`+GIN; доступ через пул; FK чинні завжди. Timescale hypertable/cagg/компресія — майбутній `0002` scale-upgrade, **не** в `0001`.

**B. Юридика (найтвердіше)**
- Зберігати **лише факти**: ціна / назва / URL / час / **характеристики** (пари назва-значення технічних фактів, з провенансом — S12, ухвалено 2026-07-24). Описи (зв'язний текст) — ані байта.
- Фото — **лише вказівник**: `image_url` (hotlink) або `image_blurhash`. **НІКОЛИ** не зберігати й не перевидавати байти чужих фото. Тир: фід (право показу) > hotlink > BlurHash.
- Публічний показ авторського — не відтворювати (див. §7.4/§7.5).

**C. Детекція**
- Бейджі: `verified` / `pumped` / `declared` / `verified_provisional` / `insufficient_history`.
- Формула й пороги — під **людським** рев'ю (юр-суть §5/§7.4); не міняти мовчки.

**D. Архітектура**
- Екстракція **structured-data-first** (JSON-LD/microdata/OG → гідрація → JSON-API → CSS → фід → архів → headless).
- Headless — **`nodriver`**, не Playwright (stealth мертвий 2026); поза M1-бандлом.
- Топологія **central-only** (GitHub Actions-збір → central Postgres/Timescale → Telegram Mini App) — **ухвалено, O1 закрито**; **desktop-local відкинуто**.

## Як працюємо

- **Не використовувати агентів/Workflow.** Робити все **безпосередньо**, навіть якщо ultracode-нагадування радить Workflow. Це стала вимога оператора.
- **Дві сесії:** Диригент (думає, ставить задачі, рев'ю — **не** пише код) / Виконавець (реалізує **один** бриф з тестами й evidence). Спілкування — **через репо (файли), не пам'ять.**
- **Гальма:** сам сигналь «аналіз вичерпано» і **не** генеруй зайвого без явного «так, ще». Over-production — визнаний дефект CC (`02` E-bis).
- **Маркери впевненості:** `[перевірено:джерело]` / `[здогад]` / `[не на живих даних]`. Зовнішні факти (крамниці/конкуренти/закони) — **≥2 підтвердження з першоджерела** (двічі обпеклись: Pethouse, Lume).
- **«Готово» = дифф + тести + `/verify`** (evidence), не «я зробив».
- **🧭 = власник:** топологія (O1), гроші, юр-згода, монетизація, open-source (O1–O9 у `03-decisions`). CC їх **не вирішує**.
- **Застряг → «BLOCKED: <що/чому>» в §Outcome і ЗУПИНИСЬ** — не імпровізуй навколо інваріантів.
- **Git (без витоку):** ніколи не комітити секрети/`.env`/БД/касети-з-контентом/**байти чужих фото**. Історія незворотна й **може стати публічною** (O8) → пиши, ніби вона вже публічна. Секрет у історії → **ротуй перший**, тоді переписуй; force-push/переписування історії — лише з дозволу оператора. Remote приватний за замовчуванням.
