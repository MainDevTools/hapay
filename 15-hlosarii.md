# Розділ 15. Глосарій

Єдина точка визначень доменних термінів (усунення розсіяних інлайн-визначень). Формат: **термін** — коротке визначення (§посилання на канон).

## Збір

- **discovery (поверхня)** — краул сторінок акцій крамниці; знаходить нові знижені товари, ставить `announce_date`; каденс 2×/добу (§3.2, §10.1).
- **baseline (поверхня)** — краул звичайних лістингів відстежуваних категорій; дає ціни й не-знижених товарів як базу 30-денної верифікації; каденс 1×/добу (§3.2, §10.1).
- **каскад** — впорядкована послідовність методів здобуття ціни з фолбеком: структуровані дані → гідратація → JSON-API → CSS → фід → архів → headless (§4.1).
- **тир A/B/C** — складність фетчу: A плейн-SSR, B cookie-challenge (Horoshop), C потрібен браузер/реверс XHR (§3.5).
- **Horoshop-родина** — крамниці на CMS Horoshop (8000+ в UA); ділять fetch-механізм і структурований метод → один адаптер-клас (§3.6).
- **probe-скрипт** — інструмент у репо, що періодично перевіряє поверхню крамниць (HTTP-код, наявність цін/JSON-LD); прототип sanity-гейта (§3.9).
- **external_ref** — стабільний ідентифікатор товару в межах крамниці, виведений з **канонізованого** URL (без tracking-параметрів); ключ `UNIQUE(source_id, external_ref)` (§4.8, §6.3).
- **preferred_method** — кешований метод каскаду, що спрацював останнім; re-probe лише при провалі (§4.1, §6.2).
- **backfill / `is_backfill`** — ретро-набудова історії з архіву (Wayback); для зоо-ніші покриття ≈0 (§4.6).

## Детекція

- **Стадія A (заявлена)** — знижка з того, що показала крамниця (перекреслена ціна); працює з дня 1 (§5.1).
- **Стадія B (верифікована)** — знижка від **нашої** 30-денної історії за статутною формулою; не довіряє крамниці (§5.2, §5.6).
- **`announce_date`** — початок поточного зниження; для незаявлених інферується й **фіксується** для відкритої події (§5.2).
- **`reference_kop`** — найменша ціна за 30 днів до анонсу (статутна база ч.10); рахується лише з сирого `price_snapshot` (§5.2).
- **30-денне вікно** — період до `announce_date` у київських цивільних добах (DST-aware, §5.2, §00).
- **бейдж** — стан знижки: `declared` (▷) / `verified` (✓) / `verified_provisional` (✓⁻) / `pumped` (⚠) / `insufficient_history` (○) (§5.3).
- **provisional (попередній зелений)** — verified при `provisional_min_points` ≤ точок < `min_reference_points`; прибирає порожній старт (§5.3).
- **pumped** — заявлена знижка ≥ порога, але verified < порога (накачана ціна) (§5.3).
- **кампанія vs сходинка** — серія знижень: база до першого анонсу (сходинка, ч.11) чи нова знижка після повернення ціни на `campaign_gap_days` (§5.5).

## Сховище й процеси

- **`price_snapshot`** — append-only факт ціни (ABORT-тригери на UPDATE/DELETE); головний актив (§6.3).
- **`price_daily`** — rollup день×товар (min/max/close) для графіка; **не** джерело статутного `reference_kop` (§6.3, §8.6).
- **`scan_run` / surface** — один прохід збору по джерелу; `surface` ∈ discovery/baseline (§6.3).
- **`detect_pass`** — обов'язковий крок після кожного `scan_run`: рахує бейджі, upsert `discount_event` (§8.4).
- **DB-пул** — єдина точка доступу до central PostgreSQL/TimescaleDB (з `statement_timeout`/`lock_timeout`); FK у Postgres чинні завжди (§6/§8.10.1). *(У відкинутому desktop-плані був SQLite connection-factory з per-connection PRAGMA — §8.2-застаріле.)*
- **worker-queue** — модель збору: конкурентно між хостами, серіалізовано per-host; один writer у БД (§8.2).
- **heartbeat / watchdog** — маркер живості collector-а + Windows Scheduled Task, що перезапускає мертвий процес (§8.2).
- **`clock_suspect`** — позначка скану під неправдоподібним системним часом (звірка з `Date`-заголовком); не живить `reference_kop` (§8.2).
- **канал adapter-config (§8.9)** — підписані декларативні дані (селектори/URL) окремим каналом; доставка фіксів без релізу.

## Робастність

- **hard-break** — 0 цін / CAPTCHA / 403 → freeze (§10.9).
- **soft-break** — адаптер щось парсить, але значення хибні (selector-drift) → канарки/аномалії (§10.9).
- **канарка (`canary`)** — 3–5 відомих товарів з очікуваним діапазоном ціни; вихід за діапазон → freeze (§10.9).
- **`frozen_at` (freeze/kill-switch)** — durable-стан «джерело замерзло»; не оновлюємо ціни, не рахуємо бейджі (§10.3).
- **probe-джоб** — окремий низькочастотний планувальник авто-розморозки замороженого джерела (§10.3).

## Конвенції та право

- **`*_kop` / копійки** — гроші лише цілим INTEGER; таблиці з грошима `STRICT` (§6.1, §00).
- **Omnibus (ч.7–11 ст.11 №3153-IX)** — спляча норма «звичайна ціна = найменша за 30 днів»; активація = кінець ВС + 1 рік (§7.2).
- **sui-generis (ст.21 №2811-IX)** — право особливого роду на неоригінальні БД, 15 років; окремі факти-ціни не охороняються (§7.4).
- **watchlist** — локальний список стеження користувача (категорія/товар/запит); не PII, не залишає машину (§6.5, §7.7).
